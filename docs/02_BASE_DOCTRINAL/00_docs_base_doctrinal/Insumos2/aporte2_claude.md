DECISIÓN 1: ESTRUCTURA DE DATOS
USAREMOS HÍBRIDO (Neo4j + YAML):
¿Por qué?

Neo4j: Para el grafo de relaciones (claims, dependencias, trazabilidad)
YAML: Para definición humana-legible de cada servilleta
JSON: Para export/import entre sistemas

Flujo:
YAML (humanos escriben) 
  → Parser 
  → Neo4j (máquinas consultan/validan) 
  → JSON (agentes consumen)
Ventajas:

Neo4j permite queries como: "Muéstrame TODAS las proposiciones que dependen de 'Dios es santo'"
YAML permite edición manual sin romper sintaxis
JSON permite integración con cualquier herramienta IA


DECISIÓN 2: ONTOLOGÍA DE RELACIONES (Tipos fijos)
7 tipos de relación canónicos:
yamlTIPOS_RELACION:
  FUNDAMENTA:      # C1 fundamenta C2 (base lógica)
  IMPLICA:         # C1 implica C2 (consecuencia necesaria)
  GENERA_PROBLEMA: # C1 genera problema que C2 resuelve
  REQUIERE:        # C1 requiere C2 para ser verdadero
  CONTRASTA:       # C1 contrasta con C2 (pero ambos son verdad)
  EJEMPLIFICA:     # C1 es ejemplo específico de C2
  CUMPLE:          # C1 cumple profecía/tipo de C2
Esto permite que agentes razonen sobre el grafo.

DECISIÓN 3: GRANULARIDAD ATÓMICA (Regla de oro)
Una proposición es atómica SI:

No puede dividirse sin perder significado
Tiene exactamente 1 predicado principal
Se puede negar sin contradicción interna

Ejemplos:
✅ ATÓMICA: "Dios es eterno"
✅ ATÓMICA: "Cristo murió como sustituto"
❌ COMPUESTA: "Dios es eterno, por tanto no cambia"
❌ COMPUESTA: "Cristo murió y resucitó"
Regla: Si tiene "y", "por tanto", "porque" → dividir.
