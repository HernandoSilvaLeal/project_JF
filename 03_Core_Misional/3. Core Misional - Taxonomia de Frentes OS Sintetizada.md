03_Core_Misional — Taxonomía de Frentes OS (Sistema Operativo Misional)

Idea nuclear: Este Core define cómo nombramos el dolor humano (A), a quién le está pasando (B), qué mecanismo lo sostiene (C), con qué herramienta intervenimos (D) y cómo lo empaquetamos sin deformarlo (E).
Es el “sistema operativo” que permite crear contenidos, campañas y rutas de discipulado sin perder claridad cuando cambian las redes o aparecen problemas nuevos.

1) Propósito y alcance
1.1 Propósito

Hacer que el proyecto pueda:

Clasificar cualquier tema sin confusión conceptual.

Diseñar series y campañas coherentes (contenido corto, largo, comunidad, herramientas).

Conectar misión con servicios (S1…S8) sin diluir el centro doctrinal.

Escalar producción con IA sin perder precisión ni humanidad.

1.2 Alcance

Este Core cubre la taxonomía misional (frentes del dolor + diagnóstico operativo + vehículos + empaquetado).
No reemplaza:

Core doctrinal (qué creemos / cómo interpretamos / 5 solas, etc.).

Core de servicios (ruta S1…S8).

Core de canales (estrategia específica por plataforma).
Pero los conecta como un “bus” de interoperabilidad.

2) Principio rector A–B–C–D (+E)

La taxonomía separa cuatro cosas que la gente suele mezclar:

A (Qué duele): frentes universales del dolor (territorios del problema).

B (Quién): sujetos/roles (afectado, causal, aliado).

C (Cómo se sostiene): mecanismos/patrones (motores del daño).

D (Con qué lo atacas): vehículos de intervención/creación (arte, educación, comunidad, ley, tech…).

E (Cómo lo empaquetas): plataformas/formatos (capa externa, adaptable).

Regla de oro:
Si algo “no encaja”, casi siempre estás intentando meter B o E dentro de A, o un elemento de C dentro de B.

3) Entidades canónicas y nomenclatura
3.1 “A* y A+”

A* = 1 frente principal.

A+ = 0–2 frentes secundarios.

Si te salen 4–6 frentes, estás describiendo un fenómeno-sistema: divide en piezas.

3.2 “F1…F5” (macro-frentes)

Son etiquetas operativas para el zoom-in: ayudan a manejar el sistema con 5 puertas principales sin borrar A1…A20.

Se usan para canales (p. ej., 1 TikTok por frente) y para autoclasificación en el Hub.

6) EJE B — Sujetos/Audiencias (Quién)

Regla operativa B (obligatoria): siempre define 3 roles:

Afectado (quién sufre)

Causal (quién produce el daño)

Aliado posible (quién puede ayudar/transformar)
Esto evita culpa general o victimismo sin salida.

10) Procedimiento estándar (7 pasos) — Producción sin romper el sistema

Pipeline canónico de una pieza (para humanos y para agentes IA):

Define el tema (1 frase).

Asigna A (A* principal + A+ secundarios 0–2).

Define B en 3 roles (afectado/causal/aliado).

Diagnostica C (1–3 mecanismos).

Elige D (1 vehículo si es pieza única; 2–4 si es campaña).

Empaqueta con E (plataforma + formato).

Define objetivo y CTA (qué cambio buscas y qué acción pides).

11) Integración con el Hub (Web/SaaS) y “no perder leads”
11.1 Hub como “fuente de verdad”

Se acordó que el Hub (sitio web / SaaS) sea:

fuente de verdad,

autoclasificación por frentes,

acceso a recursos,

buzón de oración y buzón de preguntas.

11.2 Resolver “no perder leads” con CTAs jerárquicas

En cada página del Hub:

1 CTA primaria (captura suave: email o WhatsApp)

2 CTAs secundarias:

interacción (pedir oración)

profundidad (ver enseñanza larga / YouTube)

12) Salidas (Outputs) y transiciones internas

Salidas = puertas internas entre estados del sistema (concepto tipo “eventos” en ingeniería de software). Ejemplos:

micro-CTA social: comentar/guardar/seguir (alcance, no discipulado)

CTA formal: ir al Hub (/start)

CTA de formación: entrar a Academia (S2)

CTA de cuidado: pedir consejería/discipulado (S5)

CTA de comunidad/misión: grupos (S7) o multiplicación (S6)

13) Guardrails: ética, crisis, triage

Este Core toca dolor real. Por tanto:

No sensacionalismo. El hook jamás puede ser “morbo”; debe ser “verdad + esperanza + acción”.

Crisis sensibles (A6, A2, violencia doméstica, etc.): siempre incluir derivación responsable (buscar ayuda local/profesional/emergencias cuando aplique).

0_baseChat

E no manda: ningún formato puede forzarte a trivializar el tema (E empaqueta; no redefine).
